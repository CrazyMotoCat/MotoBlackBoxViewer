## Структура решения

- `MotoBlackBoxViewer.App` — WPF UI, встроенная карта, окно, ViewModel
- `MotoBlackBoxViewer.Core` — модели, CSV-парсер, анализ телеметрии
- `MotoBlackBoxViewer.Tests` — реальные unit-тесты для парсинга и статистики

# MotoBlackBoxViewer

Стартовый каркас Windows-приложения на **C# + WPF** для просмотра логов ESP32 с мотоцикла.

## Что уже есть
- загрузка CSV с разделителем `;`
- поддержка русских заголовков
- попытка чтения UTF-8, затем CP1251
- таблица точек
- графики скорости и угла наклона
- график ускорений `X / Y / Z`
- встроенная карта маршрута прямо в окне приложения через WebView2
- подсветка выбранной точки одновременно в таблице, на графиках и на карте
- ползунок воспроизведения маршрута
- кнопки пошагового перемещения и авто-проигрывания точек
- выбор **скорости воспроизведения**: `0.25x / 0.5x / 1x / 2x / 4x`
- **фильтр диапазона точек** с обновлением карты, графиков, таблицы и статистики
- корректный расчет дистанции **для выбранного диапазона**, а не только для всего файла
- экспорт маршрута в HTML-карту и открытие во внешнем браузере

## Формат CSV
Ожидаются колонки:
- `широта`
- `долгота`
- `скорость`
- `ускорение по Z`
- `ускорение по X`
- `ускорение по Y`
- `Угол наклона`

Также поддерживаются краткие alias-имена вроде `lat`, `lon`, `speed`, `ax`, `ay`, `az`, `lean`.

Регистр не критичен.

## Как работает карта
- Встроенная карта внутри окна использует `WebView2`.
- HTML-шаблон карты лежит в `Assets/Map/index.html`.
- Маршрут передается в JS из WPF и рисуется как polyline.
- Выбор строки в таблице, движение ползунка или фильтр диапазона подсвечивают тот же участок и ту же точку на карте и графиках.
- Для подложки карты используется OpenStreetMap, поэтому для тайлов нужен интернет.


## Архитектурный рефакторинг v8
- `MainWindow.xaml.cs` уменьшен с **591** строки до **13** строк: окно больше не является центром всей логики.
- Карта вынесена в `Controls/MapViewControl`.
- Ручная отрисовка графиков вынесена в `Controls/LineChartControl`, `Controls/MultiLineChartControl` и `Controls/ChartRenderHelper`.
- Таймер воспроизведения вынесен в `Services/PlaybackService`.
- Выбор CSV вынесен в `Services/FileDialogService`.
- Управление действиями окна переведено на `Command`-подход через `RelayCommand` и `AsyncRelayCommand`.

Новая структура UI-слоя:
- `Controls` — изолированные визуальные компоненты карты и графиков
- `Services` — playback и диалог выбора файла
- `ViewModels/MainViewModel` — orchestration состояния и команд
- `MainWindow` — только компоновка окна и привязка `DataContext`
